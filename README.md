# Product Training Sandbox 🚀

Welcome to the **Product Training Sandbox** — a practice repository for learning the end-to-end PM workflow using Git and GitHub.

---

## Project Goals

- Understand how **GitHub Issues** translate client requests into actionable tasks
- Practice **branching and pull requests** to safely propose changes
- Write clear **technical documentation** that both engineers and stakeholders can read
- Learn the full **lifecycle of a feature**: from Issue → Branch → PR → Merge

---

## Success Criteria

| Criteria | Status |
|---|---|
| Repository created with README | ✅ |
| GitHub Issue written in user story format | ✅ |
| Feature branch created | ⬜ |
| Pull Request opened and linked to Issue | ⬜ |
| PR reviewed and merged | ⬜ |

---

## Workflow Overview

```
main branch (production)
│
├── issue-1-contact-button (feature branch)
│     └── Make changes safely here
│           └── Open PR → Review → Merge back to main
```

---

## Why We Don't Test in Production

Branching lets multiple people work on **separate features simultaneously** without breaking what's already live. The `main` branch always reflects the stable, working version of the project. New features live on their own branch until they are reviewed and approved.
